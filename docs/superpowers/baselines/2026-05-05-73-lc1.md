# RED Baseline: LC1 — Done-report drift between agent file and SKILL.md

## What this baseline captures

LC1 risk: if a shipped agent file restates the Brainstormer done-report field list verbatim,
and SKILL.md later renames a field, the agent file silently diverges. The role reads its
own restated list (stale) rather than SKILL.md's authoritative list (current).

This baseline documents the drift scenario with side-by-side diffs, establishes the
current state as evidence, and justifies the GREEN rule: agent files MUST reference
SKILL.md's anchor by link (D1, F6) and MUST NOT restate field lists.

---

## Scenario: field rename in SKILL.md without updating a restating agent file

### Current SKILL.md `### Brainstormer done report` (lines 373–383, commit `9fa94f8`)

```markdown
### Brainstormer done report

- `design_doc_path`: exact path to the written design doc
- `ac_ids[]`: ordered list of active AC IDs
- `intent_summary`: concise summary of what the artifact changes or decides
- `requirements[]`: full requirement set currently under review
- `adversarial_review_status`: `clean` | `findings dispositioned` | `blocked`
- `reviewer_context`: `fresh subagent` | `parallel specialists` | `same-thread fallback`
- `adversarial_review_findings[]`: findings relevant to approval, with `source`, `severity`, `location`, `finding`, and `disposition`
- `clean_pass_rationale`: required with checked dimensions when no blocker or material findings remain
- `handoff_commit_sha`: commit containing the design artifact used for approval and planning
```

### Hypothetical: SKILL.md author renames `handoff_commit_sha` → `design_handoff_sha`

**SKILL.md after edit:**

```diff
-  - `handoff_commit_sha`: commit containing the design artifact used for approval and planning
+  - `design_handoff_sha`: commit containing the design artifact used for approval and planning
```

**A restating agent file (NOT the GREEN pattern — shown as the failing case):**

```markdown
<!-- inside a hypothetical skills/superteam/.claude/agents/brainstormer.md body -->

## Done-report contract (WRONG — restates fields verbatim)
- `design_doc_path`: exact path to the written design doc
- `ac_ids[]`: ordered list of active AC IDs
- ...
- `handoff_commit_sha`: commit containing the design artifact used for approval and planning
```

### What happens without the GREEN rule (LC1 failure mode)

The Brainstormer agent reads its own body — it sees `handoff_commit_sha`. SKILL.md now
says `design_handoff_sha`. The role emits a done report using the stale field name.
Team Lead's done-report validation accepts `handoff_commit_sha` if it is not also checking
SKILL.md directly, passing a silently wrong field name into the handoff chain.

**Side-by-side diff of the drift:**

| Source | Field name | State |
|---|---|---|
| SKILL.md (after rename) | `design_handoff_sha` | authoritative, current |
| restating agent file (stale) | `handoff_commit_sha` | stale, will be emitted by role |

The operator has no visible indicator that divergence occurred. The done-report field list
in the agent file is unversioned relative to SKILL.md; any SKILL.md edit silently breaks it.

### What happens WITH the GREEN rule (LC1 fix — AC-73-2 verification target)

The agent file body contains ONLY an anchor reference:

```markdown
## Done-report contract reference

See [done-report contracts](../../SKILL.md#done-report-contracts) in
`skills/superteam/SKILL.md` for the field set this role must populate.
This file does not restate the fields.
```

When SKILL.md renames `handoff_commit_sha` → `design_handoff_sha`, the agent file
picks up the change automatically — there is nothing to drift. The anchor reference
is the single source of truth.

---

## Conclusion

Therefore agent files MUST reference SKILL.md's anchor by link (D1, F6) and MUST NOT
restate field lists. AC-73-2 verification greps for the anchor reference:

```bash
for f in skills/superteam/.claude/agents/*.md; do
  rg -F 'SKILL.md#done-report-contracts' "$f"
done
```

Expected: one match per file. Zero matches = LC1 violation.
